# 通用投标话术模板库

> 适用于各大外包平台：实现网、程序员客栈、电鸭社区、码市等

---

## 📋 投标前检查清单

- [ ] 仔细阅读项目需求 (至少 2 遍)
- [ ] 确认技术栈匹配度 (>70%)
- [ ] 评估时间可用性
- [ ] 调研类似项目市场价
- [ ] 准备相关案例展示
- [ ] 检查个人资料完整性

## ⚠️ 投标注意事项

### ❌ 避免这些错误

1. **群发模板** - 不要复制粘贴，要针对项目定制
2. **报价过低** - 低于市场价显得不专业
3. **过度承诺** - 做不到的不要答应
4. **回复太短** - 显得不重视
5. **没有案例** - 空口无凭

### ✅ 最佳实践

1. **个性化开头** - 提到项目具体细节
2. **展示理解** - 复述需求，确认理解正确
3. **提供方案** - 简要说明怎么做
4. **给出案例** - 类似项目经验
5. **明确报价** - 价格范围 + 付款方式
6. **行动号召** - 邀请进一步沟通

## 📝 跟进策略

### 投标后跟进

**24 小时未回复**:
```
您好！昨天投递了 {项目名称}，想确认下您是否收到？
如果有任何问题或需要补充信息，随时告诉我。
期待合作机会！
```

**3 天未回复**:
```
您好！关注到这个项目还在招聘中。
我对这个项目很有兴趣，也有相关经验。
如果预算或需求有调整空间，可以进一步沟通。
祝早日找到合适的开发者！
```

### 面试/沟通准备

**常见问题准备**:
1. 做过类似项目吗？→ 准备 2-3 个案例
2. 开发周期多久？→ 给出详细排期
3. 如何保证质量？→ 说明测试流程
4. 售后维护？→ 明确维护范围和期限
5. 付款方式？→ 提出分期方案

## 🎯 投标技巧

### 提高中标率的 5 个技巧

1. **抢时间** - 项目发布 24 小时内投标，竞争少
2. **看预算** - 优先投标预算明确且合理的项目
3. **秀案例** - 有类似案例的中标率高 3 倍
4. **多沟通** - 主动邀请进一步沟通
5. **持续投** - 每天投 5-10 个，量变到质变

### 筛选优质项目

**好项目特征**:
- ✅ 预算明确且合理
- ✅ 需求描述详细
- ✅ 雇主有历史招聘记录
- ✅ 回复及时
- ✅ 项目紧急 (需要尽快开始)

**警惕这些项目**:
- ❌ 预算过低 (低于市场价 50%+)
- ❌ 需求模糊 ("做个网站"这种)
- ❌ 要求先做 Demo 再决定
- ❌ 雇主从未招聘过
- ❌ 要求免费试用/测试
