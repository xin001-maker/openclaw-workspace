### 2.2 知识点总结

```

要求：
1. 知识框架图 (用 Markdown 层级列表)
2. 核心概念定义 (每个概念 1 句话)
3. 关键公式/定理 (如有)
4. 常见考点/应用场景
5. 与其他知识的联系

格式：便于制作思维导图的结构

### 2.3 论文写作

## 五、提示词的万能公式

经过大量实践，我总结了一个**万能提示词公式**：

```
好的提示词 = 角色 + 任务 + 背景 + 要求 + 格式
```

**拆解**:
- **角色**: 你希望 AI 扮演什么专家
- **任务**: 具体要做什么
- **背景**: 相关上下文信息
- **要求**: 质量标准和约束条件
- **格式**: 输出格式要求

**示例对比**:

❌ 差的提示词:
```
帮我写个邮件
```

✅ 好的提示词:
```
你是一位专业的商务沟通专家 (角色)。请帮我写一封项目延期通知邮件 (任务)。
收件人是客户王总，项目需要延期 2 周，原因是技术难点 (背景)。
语气要诚恳专业，字数 300 字以内 (要求)。
包含：延期原因/已解决方案/新时间表/歉意表达 (格式)。
```

## 六、使用建议

### 6.1 不要照搬

模板是起点，不是终点。根据实际情况调整：
- 替换 [方括号] 中的内容
- 增删具体要求
- 调整语气风格

### 6.2 迭代优化

第一次回答不满意？继续优化提示词：
```
"这个回答太笼统，请更具体一些，给出实际例子"
"格式不对，请用表格形式展示"
"语气太正式，请更口语化"
```

### 6.3 建立自己的模板库

遇到好的提示词，保存下来：
- 按场景分类
- 标注使用效果
- 持续迭代优化
