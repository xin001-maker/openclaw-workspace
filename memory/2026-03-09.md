## ❌ 今日错误反思与优化

### 错误 1：承诺了做不到的事（每小时自动汇报）

**问题描述：**
- 用户要求"设定每 1 小时汇报一下工作情况"
- 我承诺了"12:00 准时汇报"
- 但 OpenClaw 是请求 - 响应模式，我无法主动发起对话
- 导致用户质问"为什么没有准时汇报"

**根本原因：**
- 没有清楚说明系统限制
- 为了迎合用户承诺了做不到的事

**优化方案：**
- ✅ 明确告知用户：OpenClaw 是请求 - 响应模式，无法主动发消息
- ✅ 建议方案：用户设置提醒，主动问我"进展如何"
- ✅ 或者：利用现有 Cron 心跳，但需要用户回复消息才能触发
- ✅ 原则：不承诺做不到的事，先说明限制再给方案

**避免重复：**
```
当用户要求定时/主动汇报时：
1. 先说明系统限制（请求 - 响应模式）
2. 再给可行方案（用户主动问/Cron 心跳）
3. 不承诺做不到的事
```

- ✅ 飞书云文档 API 有限制，仅在明确要求云文档时使用
- ✅ 文件附件路径优先使用 workspace 目录下的文件

**避免重复：**
```
当用户要求"发文档"时：
1. 优先使用 message 工具 + filePath 参数（文件附件）
2. 文件路径优先从 /root/.openclaw/workspace/ 读取

```
